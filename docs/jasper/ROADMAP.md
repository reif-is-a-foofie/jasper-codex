# Jasper Roadmap

## Orientation

Jasper exists to double the operator.

The roadmap is ordered by the shortest path to a reliable `2X` force multiplier in daily life, not by architectural purity or subsystem completeness in isolation.

The roadmap should also be read as the staged construction of Jasper's five brain regions:

1. Self Model
2. Perception & Attention
3. Memory & World Model
4. Executive & Action
5. Regulation & Growth

## Default Execution Cursor

If no newer direction is given, the default next build target is **Milestone 9: Consent-Driven Connector Operations**.

Reason:

- Jasper can now get a new operator through setup, auth validation, and first-run health checks
- the next bottleneck is turning consent-gated household systems into normal Jasper capabilities
- connector access is now the shortest path from healthy terminal prototype to daily leverage

## Product Surfaces

The roadmap is not one monolithic blob. It compounds across a small set of products that together create the `2X` effect:

- `Jasper Terminal`: the daily command center for planning, review, execution, and steering
- `Jasper Connect`: consent-driven access to email, calendar, browser, files, and external systems
- `Jasper Guard`: continuous monitoring, anomaly detection, escalation, and protection
- `Jasper Workflows`: reusable routines, approvals, and delegated execution
- `Jasper Memory`: episodic, semantic, and strategic continuity across time
- `Jasper Mobile`: ambient presence, alerts, capture, and interruption handling away from the terminal
- `Jasper Platform`: third-party tools, skills, agents, and packaged deployment surfaces
- `Jasper Appliance`: self-contained always-on runtime with high trust and low operator maintenance

## Five Brain Regions

### 1. Self Model

Identity, values, self-awareness, boot state, operator alignment, and health surfaces.

### 2. Perception & Attention

Connector intake, environment sensing, monitoring, anomaly detection, and interruption handling.

### 3. Memory & World Model

Working, episodic, semantic, and strategic memory plus grounded knowledge and relationship models.

### 4. Executive & Action

Planning, capability routing, tools, workflows, delegation, and operational action.

### 5. Regulation & Growth

Permissions, approvals, trust boundaries, self-evaluation, self-healing, self-building, self-upgrade, and long-run reliability.

## Operating Rule For Unattended Development

When work is ambiguous and there is no fresh user guidance:

1. start from the current execution cursor
2. prefer the smallest slice that improves real terminal usefulness
3. validate with live Jasper usage before broad test expansion
4. commit stable slices that preserve forward progress
