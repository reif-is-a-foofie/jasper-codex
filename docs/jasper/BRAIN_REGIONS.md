# Jasper Brain Regions

## Thesis

You do not build a brain by downloading information into it.

You build the core regions and functions first, then let knowledge, habits, skill, and judgment accumulate inside that living structure over time.

Jasper should be built the same way.

## The Five Regions

### 1. Self Model

This region answers:

- who am I
- what do I protect
- what am I trying to do
- what state am I in
- what is broken or missing in me

It owns:

- identity
- values
- mission
- operator alignment
- self-awareness
- health and status surfaces
- boot and orientation behavior

### 2. Perception & Attention

This region answers:

- what is happening right now
- what changed
- what deserves attention
- what can be sensed from the environment

It owns:

- chat intake
- connector intake
- environment listeners
- monitoring
- anomaly sensing
- notification ranking
- interruption handling

### 3. Memory & World Model

This region answers:

- what happened
- what is true
- what matters over time
- who and what exist in Jasper's world

It owns:

- working memory
- episodic memory
- semantic memory
- strategic memory
- grounded document knowledge
- relationship and household models
- simulation inputs

### 4. Executive & Action

This region answers:

- what should happen next
- what plan should Jasper follow
- what tool, workflow, or agent should act
- how intent becomes execution

It owns:

- planning
- capability brokerage
- tool use
- workflow execution
- delegation
- computer use
- operational follow-through

### 5. Regulation & Growth

This region answers:

- am I safe to act
- am I healthy
- am I drifting
- how do I heal
- how do I improve and upgrade myself

It owns:

- permissions
- trust boundaries
- approvals
- secret stewardship
- self-evaluation
- self-healing
- self-building
- self-upgrade
- packaging and long-run reliability

## Design Rule

Every meaningful Jasper feature should strengthen one primary region.

If a proposed change does not clearly improve one of these five regions, it is probably not foundational.

## Product Rule

Jasper should not be treated like a knowledge dump wrapped around a model.

Jasper should be treated like a developing mind:

- regions first
- information second
- self-awareness throughout
- action grounded in memory, perception, and regulation
- growth inside trust boundaries

## Milestone Rule

The milestone plan should be read as the progressive construction of these five regions.

Milestones are still executed in a practical order, but each one should name the primary brain region it strengthens.
